# v3 旧版可复用材料提炼

更新时间：2026-06-03

目的：在删除初版 `01_车载系统_10kW_GZS60` 和 v2 `01_车载系统_10kW_GZS60_v2` 前，把对 v3 后续建模仍有价值的内容提炼到 v3 内部，避免旧目录继续污染主线。

## 1. 总体判断

v3 应继续作为唯一建模研究主线。初版和 v2 的价值不在于继续复跑旧模型，而在于提供以下三类信息：

| 类别 | 处理方式 |
|---|---|
| 可复用参数和处理后数据 | 已复制到 `00_输入参数/旧版提炼` |
| 可复用建模纪律、接口经验和验证教训 | 已在本文档中提炼 |
| 旧模型、旧脚本、旧结果体系 | 不迁入 v3，不建议继续引用 |

删除旧目录后，v3 不应再跨目录读取初版或 v2 的脚本、模型、结果和参数。若后续确实需要旧版某个公式或参数，应从本文档或 `00_输入参数/旧版提炼` 出发，重新登记到 v3 台账。

### 2.2 可复用结论

- 初版已确认当前自吸 cEGR 结构主路径：新鲜空气/EGR 回流 -> 空气机 -> 中冷器 -> GZS60 干侧 -> 电堆 -> 背压阀 -> GZS60 湿侧 -> 水气分离器 -> EGR/排气分流。
- DQ60 在当前阶段只能作为气路边界执行器和 MAP 可达性参考，不能支撑空压机功耗、净功率和系统效率结论。
- GZS60 参数主要来自系列趋势、说明书约束和 GZS80P 实测辅助，不能声明已经完成 GZS60 直接标定。
- 固定电流、固定电压、恒入堆氧分压等结果应作为后处理目标，不应反向改变标准模型主拓扑。
- BOP 辅件需要明确分项可信度标志：空气机 MAP 数字化、阀件默认、分水器默认、中冷器默认、GZS60 低流量外推、旁通忽略。

### 2.3 不再迁移的内容

- 初版 `sim_metrics` 和大量监控指标不再迁入。v3 输出应继续保持节点表、系统摘要、验证摘要和少量必要诊断。
- 初版 GUI、监控面板、PPT 生成、批量 sweep、专项审计脚本不再作为 v3 工作流。
- 初版历史结果只保留经过提炼的数据和结论，不再保留结果文件体系。

### 3.2 v2 必须继承的建模纪律

- 不沿用初版 645 维匿名状态向量。
- 不沿用初版 288 项 `sim_metrics`。
- 模块之间只传必要物理接口，不传全局大状态。
- 每个物理量必须有唯一所有者方程，其他模块只能读取，不能重复覆盖。
- 核心状态、派生量和诊断量必须分离；诊断残差和风险标志不得回写主物理状态。
- 长期结果只保留节点表、系统摘要和验证摘要，不长期沉积 CSV、PNG、PPT 或过程审计文件。

这些纪律已经与 v3 当前方向一致，应继续保持。

### 3.3 v2 可复用方程口径

#### 电堆库存和压力

电堆内部压力由库存质量、体积和温度通过理想气体状态方程反算：

```text
p_i = n_i R T / V
n_i = m_i / M_i
p_ca = pO2_ca + pN2_ca + pH2Ov_ca
```

v3 后续不能直接指定电堆内部压力。上游或下游压力只能作为边界，库存压力必须由电堆状态反算。

#### 出流更新

v2 修正步长敏感性的关键做法是：

```text
m_i,pre = m_i,k + dt * source_i
先做水蒸气饱和裁剪
p_pre = f(m_i,pre, T)
m_out,total = K * (p_pre - p_back) / (1 + K * A * dt)
m_i,out = w_i,pre * m_out,total
m_i,k+1 = m_i,pre - dt * m_i,out
末态再做饱和裁剪
```

对 v3 的含义：

- 出流应由压力差驱动，而不是由上游流量强行分配。
- 出流组分应按源项预更新后的库存质量分数分配。
- 饱和裁剪应在源项更新后和末态更新后都检查。
- 若后续 v3 增加动态库存，应优先采用这种半隐式出流口径或局部迭代，不优先放宽容差。

#### 膜水传输

v2 的准静态膜水口径仍可作为 v3 第一阶段增强方向：

```text
a_an = pH2O_an / p_sat(T_stack)
a_ca = pH2O_ca / p_sat(T_stack)
lambda_eq = f(a)
lambda_mem = 0.5 * (lambda_eq_an + lambda_eq_ca)
C_an = rho_mem / EW * lambda_eq_an
C_ca = rho_mem / EW * lambda_eq_ca
J_H2O_mem = n_d * I / F - D_mem * (C_ca - C_an) / delta_mem
m_H2O_mem = M_H2O * A_cell * N_cell * J_H2O_mem
```

膜水通量必须作为阴极和阳极水蒸气守恒的成对源项，不应只作为电压经验修正。

#### 两种阴极流量控制模式

v2 明确区分两类目标，v3 后续扩展时也应保留：

| 模式 | 含义 | 用途 |
|---|---|---|
| `fixed_operating_target` | EGR 开启后保持同一总入口流量口径，新鲜空气被回流气挤占，入口氧分压下降 | 研究 EGR 稀释、低负荷高电位限制和自加湿趋势 |
| `fixed_ca_in_pO2` | 通过提高空压机总入口流量维持入堆氧分压 | 区分“氧分压保持”与“EGR 自加湿/热湿效应” |

这两类模式不能混用结论。固定入口氧分压不是固定氧气摩尔分数，也不是固定电堆内部库存氧分压。

### 3.4 v2 验证教训

- v2 第一版步长敏感性失败的根因是库存更新和压力出流关系不够稳健。
- v2 第二版 Gate 1 通过主四工况、600 s 长时间检查、`dt_s = 1.0 / 0.5 / 0.25 s` 步长收敛、物种残差和 EGR 无液态水约束。
- v2 第二版 Gate 2 通过 Simulink 六工况一致性，但这只作为历史经验，不作为 v3 当前验证证据。
- v3 继续以自身验证结果为准，不能把 v2 通过状态写成 v3 通过状态。

## 5. 删除旧目录前的结论

从 v3 后续建模角度看，初版和 v2 目录可以删除，前提是接受以下边界：

- 不再复跑初版或 v2 模型。
- 不再保留初版 GUI、PPT、批量 sweep 和历史图件。
- 不再保留 v2 历史验证 CSV/MD 原文件。
- 后续所有建模、仿真、验证、说明和参数登记只在 v3 内进行。

已保留的材料足以支持 v3 后续继续推进：

- DQ60 MAP 数字化资料已在 v3 保留。
- 台架数据接口已在 v3 保留。
- BOP/GZS 参数来源已在 v3 保留。
- GZS 通用化数据和参数反演已在 v3 保留。
- v2 关键建模纪律、守恒更新方式和验证教训已在本文档保留。

## 6. 后续 v3 采用建议

下一步不建议继续从旧目录“找代码改”。建议按以下顺序推进：

1. 先把新书《质子交换膜燃料电池系统及其控制》的阴/阳极集总流量、膜水合和 BOP 方程映射到 `v3_模块方程与源项台账.md`。
2. 再把 v2 半隐式库存出流和膜水成对源项作为 v3 电堆动态增强候选。
3. 然后接入 `00_输入参数/旧版提炼/空气机_DQ60_MAP数字化`，但只作为 MAP 可达性和外推诊断，不直接输出系统效率结论。
4. 最后用 `00_输入参数/旧版提炼/GZS通用化数据与参数反演` 增强 GZS60 加湿器模型，并保留 GZS60 低流量外推标志。
