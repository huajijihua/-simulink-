## 3. 总体建模目标

v2 第一版优先保证物理一致性，而不是复现旧 v1 扫点曲线。

验收重点：

- 物质流清楚：`O2 / N2 / H2O_v / H2O_l / H2` 的来源、去向、反应、相变和分流可追踪。
- 压力流清楚：总压、分压和压降由明确方程或边界得到，同一物理量不应多处覆盖。
- 能量流清楚：空压机升温、中冷器降温、加湿器水热交换、电堆产热散热和尾气温度路径不互相矛盾。
- 文件精简：只保留最小闭环运行、检查、说明和验证输出。

## 4. 模型拓扑

顶层按模块函数组织，不把每个公式拆成大量 Simulink 小块。

主链路：

```text
Environment/FreshAir
-> EGRMixer
-> Compressor
-> Intercooler
-> HumidifierDry
-> PEMFCStack
-> BackPressure/Outlet
-> HumidifierWet
-> Separator
-> EGRSplitter
-> recycle/vent
```

各模块之间只传递最核心变量。诊断量由监测/验证脚本计算，不反向参与主方程。

## 5. 状态与接口口径

第一版接受“命名状态向量”作为过渡，不直接上完整 Bus/Data Dictionary。

要求：

- 禁止大面积裸写 `state(95)` 之类匿名索引。
- 用命名索引常量或结构化读写函数表达核心变量。
- 核心状态和诊断量分离。
- 不兼容旧 `sim_metrics`、旧入口脚本名和旧结果字段名。

必要动态状态只保留有物理意义的库存/温度：

- 阴极通道库存：`m_O2_ca, m_N2_ca, m_H2O_v_ca, m_H2O_l_ca`
- 阳极通道库存：`m_H2_an, m_H2O_v_an, m_H2O_l_an`
- 膜含水状态：`lambda_mem` 或等效膜水量
- 电堆本体温度：`T_stack`
- 必要时可加入加湿器干/湿侧一阶滞后状态，但不能扩展成庞大诊断状态。

## 6. 外界环境与入口气体

外界环境以三种气体物质为基础：

- `O2`
- `N2`
- `H2O_v`

主输入：

- `T_amb`
- `p_O2_amb`
- `p_N2_amb`
- `p_H2O_amb`

便捷输入可支持：

- `T_amb`
- `p_amb`
- `RH_amb`

便捷输入必须在初始化或入口模块中转换成三种气体分压，再统一转换为质量流。

## 7. 空压机

空压机第一版作为边界执行器/热压升模块。

要求：

- 负责总入口流量、压力升高、温度升高。
- 不改变 `O2 / N2 / H2O` 的质量和组分。
- 功耗可输出为诊断，但不作为第一轮结论。
- 可使用简单等熵/效率温升模型，或给定压升和温升边界。
- DQ60 MAP 数据可作为压升/功耗诊断或边界限制来源，但不迁移旧空气机审计脚本。

建议迁移的必要空气机数据：

- `air_machine_DQ60_digitized_dp_map_v01.csv`
- `air_machine_DQ60_digitized_power_map_v01.csv`
- `air_machine_DQ60_pdf_axis_digitized_points_v02.csv`

## 8. 中冷器

中冷器第一版只做温降、压降和水相变，不建冷却液动态回路。

要求：

- `O2`、`N2` 质量流不变。
- 总水量 `H2O_v + H2O_l` 守恒。
- 出口温度低于空压机出口。
- 出口压力低于入口压力。
- 若水蒸气超过出口温压下饱和容量，超出部分液化。
- 冷却液侧只作为边界，可用给定出口目标温度或简化换热效率/UA。

## 9. 加湿器

v2 必须写入约束：湿侧水只来自电堆阴极尾气，干侧不允许外部补水。

干侧：

- `O2`、`N2` 守恒。
- `H2O` 增量来自湿侧传水。
- 干侧出口温度由湿侧传热/传水影响。
- 干侧出口压力按简化压降计算。

湿侧：

- 湿侧入口紧接阴极出口。
- 湿侧入口温度应高于或接近干侧入口；若低于干侧入口，应输出诊断，不用经验修正掩盖。
- 湿侧水减少量应与干侧传水量闭合。
- 液态水不直接穿膜到干侧。

建议迁移的必要 GZS60 数据：

- `GZS60车载模型参数_v02.csv`
- `GZS供应商规格摘要.csv`
- `GZS加湿器数据与建模建议.md`

不迁移：

- GZS80P 拟合包
- GZS80P 回放/审计脚本
- 低流量外推扫点和图

缺失的 GZS60 参数应集中标记为待标定工程参数，不用 GZS80P 外推掩盖。

## 10. 水相变

第一版统一用饱和蒸气压/饱和含湿量裁剪，不引入复杂两相流。

规则：

- 对每个气体节点，根据 `T` 和 `p` 计算水蒸气可容纳上限。
- 若 `m_H2O_v > m_H2O_v_sat`，超出部分转为 `m_H2O_l`。
- 若有液水且气相未饱和，可按简化规则蒸发补足到饱和，但受可用液水限制。
- 相变潜热先作为诊断量；是否回灌温度方程必须在能量模型中明确，不允许隐式改温度。

## 11. 电堆质量守恒

电堆第一版应从五个主要气体质量守恒方程开始：

- 阴极 `O2`
- 阴极 `N2`
- 阴极 `H2O`
- 阳极 `H2`
- 阳极 `H2O`

阴极示例：

```text
dm_O2_ca/dt = m_O2_in - m_O2_react - m_O2_out
```

电堆内分压由库存质量、体积、温度通过理想气体状态方程反算，不能由台架边界压力直接覆盖。

电堆出堆流量第一版按线性压力-流量关系：

```text
m_ca_out = K_ca_out * max(p_ca - p_cathode_back, 0)
```

各气相组分按堆内库存质量分数流出：

```text
m_O2_out = y_O2_mass * m_ca_out
m_N2_out = y_N2_mass * m_ca_out
m_H2O_v_out = y_H2O_v_mass * m_ca_out
```

液水出口可用独立 `K_liq` 或库存排出规则处理。

## 12. 阳极侧

阳极第一版只保留入口边界和阳极通道库存，不建氢循环泵、喷氢阀、阳极循环细节。

阳极边界建议包含：

- `m_H2_in`
- `m_H2O_v_in`
- `T_an_in`
- `p_an_in`
- `p_an_back`

阳极温度第一版直接取 `T_an = T_stack`。

## 13. 膜水传输

膜水传输保留两个低维源项：

- 电渗拖曳：随电流由阳极向阴极迁移。
- 浓度梯度反扩散：由阴阳极水浓度/RH差驱动。

两者合成为膜水净源项，进入阴极/阳极水守恒。

膜含水量 `lambda_mem` 用一阶动态或等效水量平衡更新，不做膜厚方向 PDE 或分层模型。

## 14. 电堆能量守恒

电堆本体温度由热力学第一定律构建集总能量守恒。

基本形式：

```text
C_stack * dT_stack/dt =
    Q_total_generated
  - P_electric
  - Q_coolant
  - Q_ambient
  - Q_gas_absorbed
```

要求：

- `Q_total_generated` 和 `P_electric` 保留物理含义，不能混成一个经验发热项。
- 冷却液散热、环境散热、气体吸热分别作为独立项。
- 阳极温度第一版用电堆本体温度表示。
- 电压方程使用 `T_stack`。
- 阴极出入口温度和本体温度可能存在差距，但正常热态下应遵循 `T_ca_in < T_stack < T_ca_out` 的方向性。

阴极出口气体温度第一版由气体吸热项反算：

```text
T_ca_out = T_ca_in + Q_gas_absorbed / (m_ca_out * cp_gas)
```

湿侧加湿器入口温度直接使用 `T_ca_out`。

## 15. 电压模型

电压模型第一版沿用半经验结构：

```text
V_cell = E_rev - eta_act - eta_ohm - eta_con
```

要求：

- `E_rev` 使用 `T_stack`、`pH2`、`pO2`、水状态等核心变量。
- `eta_act / eta_ohm / eta_con` 只读取 v2 核心变量，例如 `j`、`lambda_mem`、`T_stack`、`pO2`。
- 不从旧 `state` 索引或台架边界压力直接覆盖电化学变量。
- 液水第一版只作为传质风险诊断，不引入未标定的液水淹没电压惩罚系数。

建议迁移的必要电堆数据：

- `full_range_polarization_data.csv`
- `stack_no_egr_physics_params.csv`
- `stack_no_egr_physics_params.mat`
- 必要推荐工况摘要或参数来源说明

## 16. EGR 与背压

EGR 分流第一版使用指定回流比例/阀开度映射，不建详细可压缩阀模型。

```text
m_EGR = egr_fraction_cmd * m_separator_out
m_vent = (1 - egr_fraction_cmd) * m_separator_out
```

背压阀第一版使用指定阴极出口/尾气压力边界或简单压降，不建详细阀模型。

电堆内部出堆流量仍由堆内压力与下游压力差决定，背压模块只提供下游压力边界。

## 17. 工况模式

v2 第一版需要支持恒电流和恒电压两种工况，但实现顺序为先恒电流，后恒电压。

恒电流：

- 直接给定 `I_stack` 或 `j_Acm2`。
- 用于物质守恒、气路、水热链路主验证。

恒电压：

- 每个仿真步内用电压方程反求电流。
- 先用简单数值求解，不引入复杂控制器。
- 与恒电流共用同一套气体库存、分压、电压、水热方程。

cEGR 核心控制目标必须保留：

- 对恒电流和恒电压工况，保持空压机总流量基本不变时，EGR 增大应降低入堆 `xO2` 和 `pO2`。
- 恒电流下，EGR 增大后电压不应升高。
- 恒电压下，EGR 增大后可维持电流不应升高。

## 18. 离散更新

v2 第一版默认使用固定步长离散更新。

建议：

- `dt_s` 默认 `0.1` 或 `1.0` 秒。
- MATLAB Function 内按显式欧拉或一阶更新核心库存/温度。
- Simulink 顶层使用离散链路和 Unit Delay 闭环。
- 这样便于逐步检查每一步物质、压力和能量收支。

## 23. 第一轮完成判据

第一轮可认为完成的条件：

- v2 模型能构建并运行。
- 恒电流和恒电压代表工况能输出节点表和系统摘要。
- 关键模块物质守恒残差在合理容差内。
- 压力方向合理：压缩机升压，后续部件压降，堆内压力与背压边界关系清楚。
- 温度方向合理：压缩机升温，中冷器降温，加湿器湿侧入口来自阴极出口，正常热态下 `T_ca_in < T_stack < T_ca_out`。
- EGR 增大后入堆 `xO2/pO2` 下降；恒电流下电压不升高；恒电压下可维持电流不升高。
- `03_说明/v2_验证记录.md` 和 `04_验证结果/v2_validation_summary.md` 已同步更新。
