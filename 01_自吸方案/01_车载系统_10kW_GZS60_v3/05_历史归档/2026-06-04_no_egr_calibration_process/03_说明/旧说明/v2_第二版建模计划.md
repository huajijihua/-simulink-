# 10 kW GZS60 cEGR v2 第二版建模计划

更新时间：2026-06-02

> 当前状态：仅作历史参考，不作为本轮 v2 建模实施依据。
>
> 本文件记录第一版暴露步长敏感性问题后的修正计划。该路线曾尝试让 MATLAB 脚本核心阶段性领先 Simulink，后续发现脚本与 Simulink 难以同步。当前项目已改用 `v2_当前建模实施计划.md`，以 Simulink 系统模型为核心重新实施。

本文档记录 v2 第二版建模前已经确认的需求对齐结论、实施边界、接口变化和验收口径。第二版继续沿用当前 v2 目录、模型名和五个脚本入口，不新建 `v3` 或 `v2_phase2` 目录。

## 1. 第二版核心目标

第二版的第一优先级是查明并修正第一版暴露出的步长敏感性失败问题。模型根基必须回到清楚的质量守恒、压力-出流关系和库存更新，而不是继续依赖库存松弛近似。

硬性完成条件：

- 主四工况在 `dt_s = 1.0 / 0.5 / 0.25 s` 下满足步长收敛容差。
- 电堆阴极和阳极质量守恒方程清楚，并能输出物种级残差摘要。
- 脚本核心通过后，同步 Simulink 模块链路，并完成脚本-模型一致性检查。
- 第一版步长敏感性失败必须修复；若未修复，第二版不能视为完成。

第二版暂不追求复杂 BOP 高保真模型。空压机、中冷器、加湿器、背压、EGR 阀和分水器只做与守恒链路相关的必要修正。

## 3. 实施门槛

第二版允许脚本核心阶段性领先 Simulink，但最终验收必须同步模型。

```text
Gate 1: 脚本核心主四工况 + 步长收敛通过
Gate 2: Simulink 六工况运行 + 与脚本核心一致性通过
```

若半隐式解析更新不能满足步长容差，下一步优先采用电堆阴极压力-出流局部小型迭代。内部子步长积分只作为诊断工具，不作为首选正式方案；不优先放宽容差。

## 4. 核心状态接口变化

第二版将 `lambda_mem` 从动态状态改为准静态派生量。核心动态状态由第一版 9 维改为 8 维：

```text
[mO2_ca, mN2_ca, mH2Ov_ca, mH2Ol_ca,
 mH2_an, mH2Ov_an, mH2Ol_an,
 T_stack_C]
```

`lambda_mem` 每一步由阴阳极水活度计算：

```text
lambda_mem = (lambda_eq_an + lambda_eq_ca) / 2
```

阳极液态水库存 `mH2Ol_an` 保留，用于阳极侧过饱和裁剪和水守恒。

## 5. 电堆压力与质量守恒重构

电堆内压力永远由库存质量、体积和温度通过理想气体状态方程计算：

```text
p_i = n_i R T / V
n_i = m_i / M_i
p_ca = pO2_ca + pN2_ca + pH2Ov_ca
```

第二版不直接指定电堆内部压力。阴极出口采用线性压力-总质量流量边界：

```text
m_ca_out,total = K_ca_out * max(p_ca - p_cathode_back, 0)
```

第二版优先采用半隐式末态压力解析近似。处理顺序：

```text
m_i,pre = m_i,k + dt * source_i
先对 mH2Ov_pre 做饱和裁剪，超出部分转入 mH2Ol_pre
p_pre = f(mO2_pre, mN2_pre, mH2Ov_pre, T)
w_i,pre = m_i,pre / sum(m_gas,pre)
m_out,total = K * (p_pre - p_back) / (1 + K * A * dt)
m_i,out = w_i,pre * m_out,total
m_i,k+1 = m_i,pre - dt * m_i,out
末态再做一次饱和裁剪
```

其中 `A` 是由当前气体组成、摩尔质量、`R*T/V` 得到的压力变化系数。若 `p_pre <= p_back`，出口总质量流量为 0。

出流组分采用总质量流量边界 + 源项预更新后的库存质量分数分配：

```text
m_i,out = w_i,pre * m_out,total
```

## 6. 膜水传输模型

第二版采用书籍资料中的准静态膜水传输模型，不引入复杂动态膜模型。跨膜水通量由电渗拖曳和反扩散组成：

```text
a_an = pH2O_an / p_sat(T_stack)
a_ca = pH2O_ca / p_sat(T_stack)
lambda_eq = f(a)
lambda_mem = (lambda_eq_an + lambda_eq_ca) / 2
C_an = rho_mem / EW * lambda_eq_an
C_ca = rho_mem / EW * lambda_eq_ca
J_H2O_mem = n_d * I / F - D_mem * (C_ca - C_an) / delta_mem
m_H2O_mem = M_H2O * A_cell * N_cell * J_H2O_mem
```

膜水通量作为阴阳极水蒸气守恒方程源项，不再通过一阶滞后积分 `lambda_mem`。

## 7. 阳极边界

阳极侧只建最简边界，不引入喷氢阀、氢循环泵、排氢排水系统或其他阳极 BOP。

```text
阳极入口边界 -> 电堆阳极库存 -> 阳极出口边界 -> 排空
```

阳极氢气入口按固定计量比动态给：

```text
mH2_an_in = lambda_H2_an * I_stack * N_cell / (2F) * M_H2
```

阳极入口水蒸气按相对湿度边界给定：

```text
RH_an_in = 0.30
T_an_in ~= T_stack
p_an_in = p_ca_in + 10 kPa
pH2O_an_in = RH_an_in * p_sat(T_an_in)
pH2_an_in = p_an_in - pH2O_an_in
mH2Ov_an_in = mH2_an_in * (pH2O_an_in / pH2_an_in) * (M_H2O / M_H2)
```

阳极出口背压固定为阳极入口压力略低：

```text
p_an_back = p_an_in - dp_an_channel
dp_an_channel ~= 3 kPa
```

阳极出口采用线性压力-总质量流量边界，并按阳极库存质量分数分配 `H2/H2Ov` 出流：

```text
m_an_out,total = K_an_out * max(p_an - p_an_back, 0)
```

## 8. 阴极流量控制模式

第二版明确区分两类 cEGR 研究目标，不能混用结论。

### 8.1 固定运行目标模式

模式名：

```text
fixed_operating_target
```

用于现有主四工况，研究 EGR 稀释氧气、降低入口/电堆氧分压和电压趋势。空压机总入口质量流量按目标氧气计量比决定：

```text
m_comp,total = m_fresh + m_EGR
```

EGR 开启后保持同一总入口流量口径，新鲜空气被回流气挤占，入口氧分压下降。

### 8.2 固定阴极入口氧分压模式

模式名：

```text
fixed_ca_in_pO2
```

固定的是加湿器干侧出口/电堆阴极入口氧分压：

```text
pO2_ca_in_kPa
```

不是固定氧气摩尔分数，也不是固定电堆内部库存氧分压。

`pO2_ca_in_kPa` 与 `pO2_ca_kPa` 的区别：

- `pO2_ca_in_kPa`：加湿器干侧出口/电堆阴极入口氧分压。
- `pO2_ca_kPa`：电堆阴极库存氧分压，由 `PV=nRT` 计算，用于电压模型。

固定入口氧分压模式下，开启 EGR 后通过提高空压机总入口质量流量维持：

```text
abs(pO2_ca_in_egr - pO2_ca_in_baseline) <= 0.5 kPa
m_comp,total_fixed_pO2_egr > m_comp,total_no_egr
```

## 9. 空压机与中冷器

空压机第二版保持简化，只作为边界设备：

- 混合外界环境气体和 EGR 回流气体。
- 提供合理的压力升高和温度升高方向。
- 支持两种阴极流量控制模式。
- 不接 DQ60 MAP。
- 不输出空压机功耗，不计算系统净功率。

中冷器保持简化调温设备：

```text
T_out = target_intercooler_T
p_out = p_in - dp_intercooler
```

中冷器必须判断湿空气降温冷凝：

```text
若 mH2Ov_in > mH2Ov_sat(T_out, p_out):
    mH2Ov_out = mH2Ov_sat
    mH2Ol_out = mH2Ol_in + (mH2Ov_in - mH2Ov_sat)
否则:
    mH2Ov_out = mH2Ov_in
    mH2Ol_out = mH2Ol_in
```

中冷器冷凝液第二版不从系统移除，作为 `mH2Ol` 随气体节点进入下游。

## 10. 加湿器模型

第二版加湿器采用简化 epsilon-NTU / 流量相关有效度模型，替代固定效率。文献依据为 PEMFC 膜加湿器集总模型中常用的 effectiveness-NTU 水热传递形式；项目数据口径为 GZS60 说明书约束设备边界，GZS80P 实测趋势辅助估计性能规律。

设备和数据口径：

- GZS60 定义设备边界、额定流量、压降、露点、体积和内漏边界。
- GZS80P 实测趋势辅助确定换湿有效度随流量变化的初始规律。
- 不声明 GZS60 加湿器已经完成定量标定。

换湿模型：

```text
NTU_hum = NTU_ref * (m_ref / m_flow)^n
epsilon_hum = 1 - exp(-NTU_hum)
m_transfer = epsilon_hum * min(dry_need, wet_available)
dry_gain = m_transfer
wet_loss = m_transfer
```

效率只限制传递量大小，不额外引入“湿侧损失大于干侧获得”的不可验证参数。后续若有 GZS60 四口实测数据，再修正该模型。

加湿器压降尝试改为流量相关：

```text
dp_dry = dp_dry_ref * (m_dry / m_ref)^n_dp_dry
dp_wet = dp_wet_ref * (m_wet / m_ref)^n_dp_wet
```

初始参数由 GZS60 说明书额定点约束，指数使用已有整理值或工程默认，并标记为待校准。

加湿器验收重点：

- 干侧不能超过饱和。
- 湿侧水只来自电堆阴极出口。
- 湿侧入口温度应高于或接近干侧入口。
- 水热交换方程要合理，但不把加湿器性能表现作为严格水热残差验收对象。

## 11. 液态水路径与 EGR 液滴保护

中冷器和加湿器干侧产生或携带的入口液态水进入电堆阴极液态水库存：

```text
dmH2Ol_ca/dt += mH2Ol_ca_in
```

阴极液态水随气流排出，进入阴极出口尾气。携液公式采用：

```text
mH2Ol_ca_out =
    K_liq_carry * mH2Ol_ca * (m_gas_out,total / m_gas_inventory)
```

液态水路径：

```text
电堆阴极液水库存
-> 阴极出口尾气携液
-> 背压边界
-> 加湿器湿侧
-> 水汽分离器
-> 排液移除
```

分水器逻辑：

1. 对湿侧出口气体按当前 `T, p` 做饱和裁剪。
2. 超饱和 `H2O_v` 转为 `H2O_l`。
3. 分水器按效率移除液态水。
4. `O2/N2/H2O_v` 随气体继续进入 EGR/排气分流。

EGR 分流数学上仍按所有剩余组分比例分流，但进入空压机入口前必须无液态水。第二版加入极简液滴保护：

```text
mH2Ol_egr_to_compressor = 0
mH2Ol_egr_removed = f_egr * mH2Ol_separator_out
```

硬性验收：

```text
mH2Ol_egr_to_compressor <= 1e-12 kg/s
```

`egr_liquid_removed_kg_s` 写入验证摘要，不新增长期 CSV。

## 13. 恒电压求解

第二版继续保留恒电压工况，并继续使用二分法求解电流，不使用 `fsolve` 或 Newton。

需要增加求解诊断：

```text
voltage_solve_ok
voltage_solve_reason
```

至少区分：

- 目标电压可达。
- 目标电压过高。
- 氧分压过低。
- 达到限制电流仍无法满足。
- 二分结果异常。

脚本 `systemSummary` 写入字符串原因；Simulink `summarySignal` 只写数值标志；Markdown 汇总原因。

## 14. 验收工况

第二版 `ScenarioIndex` 扩展到 6：

```text
1 cc_no_egr
2 cc_egr_0p4
3 cv_no_egr
4 cv_egr_0p4
5 cc_fixed_pO2_egr_0p4
6 cv_fixed_pO2_egr_0p4
```

主四工况用于正式主验收。两个固定入口氧分压工况作为扩展验收，不做 sweep。

步长收敛只对主四工况执行：

```text
dt_s = 1.0 / 0.5 / 0.25 s
```

容差：

- 恒电流：`dI_stack_A = 0`。
- 恒电压：`dI_stack_A <= 0.5 A`。
- `dV_cell_V <= 0.01 V`。
- `dpO2_ca_kPa <= 2 kPa`。
- `dT_stack_C <= 1 degC`。
- 压力方向、温度方向全部通过。
- 质量残差 `<= 1e-8 kg/s`。

长时间检查保留：

- 主四工况保留 `120 s` 与 `600 s`。
- `600 s` 检查稳定性和方向，不要求与 `120 s` 末态完全一致。
- 步长收敛在同一 `StopTime` 下比较。

固定入口氧分压扩展工况检查：

- 脚本和 Simulink 都可运行。
- `abs(pO2_ca_in_egr - pO2_ca_in_baseline) <= 0.5 kPa`。
- EGR 后所需 `m_comp,total` 上升。
- 趋势解释合理。

## 16. 参数管理

第二版重整参数分类：

```text
physical_constant
geometry_or_material
calibration_parameter
```

示例：

- `physical_constant`：`F, R, M_O2, M_N2, M_H2O, M_H2`
- `geometry_or_material`：体积、活性面积、膜厚、EW、膜密度等
- `calibration_parameter`：流量系数、换热系数、加湿器 NTU/流量指数、压降系数、液水携带系数等

GZS60 加湿器参数优先来自 GZS60 说明书或 GZS60 实测数据。GZS80P 实测只能辅助建立性能趋势和初值，不作为 GZS60 定量标定结论。
