# 电堆核心修正计划 v02：膜水动态与跨膜水通量

## 目的

本文件承接 `电堆核心修正计划_v01.md`。v01 已完成边界、源项、饱和裁剪和审计框架的第一轮修正，但详细审计显示膜水净传输质量流量 `mMem` 在全部工况末段仍存在正负波动，且低电流工况下跨膜水通量绝对值明显大于产水量和入口水蒸气量。

v02 的目标是修正膜水模块的动态口径，使其适合当前零维系统级电堆模型，而不是直接套用穿膜方向一维模型。

## 新增参考依据

### 1. 系统级综述对模型维度的区分

参考文本：

- `04_仿真结果/membrane_reference_text_02/Areviewofphysics-basedlow-temperatureproton-exchangemembranefuel cell models for system-level water andthermalmanagementstudies.txt`

关键结论：

- 0D 模型把阳极和阴极分别看作单一控制体，不解析空间分布；需要通过经验或半经验关系弥补空间细节不足。
- Pukrushpan et al. 2004 被归类为瞬态 0D、单相、等温控制导向模型，并考虑氧/氢分压、电堆温度、膜水含量对电压的影响。
- Ritzberger et al. 2020 被归类为瞬态 0D、两相、等温模型。
- Salva et al. 2016 被归类为稳态穿膜方向 1D、两相、非等温模型，不适合作为当前零维电堆核心的直接实现模板。

### 2. 膜水动态处理方式

系统级综述的膜水传输部分给出了适合本项目的判断：

- 常见准静态做法是用阴极/阳极水活度计算 `lambda_ca`、`lambda_an`，再令 `lambda_m = 0.5*(lambda_ca + lambda_an)`。
- 这种做法不解膜控制体守恒，不能复现膜吸附/解吸动态。
- Ritzberger 的 0D 思路是让膜水活度 `a_memb` 按阴阳极平均水活度的一阶时间常数滞后：

```text
d a_memb / dt = (0.5*(a_ca + a_an) - a_memb) / tau_memb
lambda_m = lambda_eq(a_memb)
```

### 3. 对当前问题的解释

当前 `mMem` 不收敛更可能是三类因素叠加：

1. 方程口径问题：当前准静态 `lambda_m` 每步直接跟随气相水活度，膜水没有自身动态缓冲。
2. 参数尺度问题：当前全堆有效膜面积乘以瞬时浓度梯度后，低电流下 `mMemRaw` 可达到产水量的数十倍，说明需要有效传输系数或吸脱附动力学约束。
3. 求解问题：显式回灌和 CEGR 延迟会放大快速源项的符号反复，若膜水源项过硬，容易形成末段正负振荡。

因此 v02 先修方程和尺度，再评估求解器设置；不应先靠求解器掩盖源项过强。

## 修正原则

1. 不直接采用 2021 自循环自增湿论文中的穿膜方向 1D 厚度分布公式作为当前零维模型实现。
2. 保留五个气体质量状态和一个温度状态的电堆核心框架。
3. 新增一个零维膜水动态状态，固定采用 `a_memb`，避免把膜水每步强制设为气相瞬时平衡值。
4. 跨膜水通量仍保留电渗拖曳和反扩散两个源项，但先以原始通量 `J` 的单位链路计算：

```text
J_drag = n_d * i / F
J_diff = D_w * c_f * (lambda_ca - lambda_an) / delta_mem
mMem_raw = (J_drag - J_diff) * A_cell * N_cell * M_H2O
```

5. `mMem > 0` 仍定义为水从阳极到阴极。
6. 增加可标定的有效跨膜传输系数 `k_mem_eff`，默认先取小于等于 1，不改变方向，只控制总尺度；第一轮只用量级和稳定性约束，不参与电压拟合。
7. `a_memb` 是零维等效膜水活度滞后状态，不作为严格膜内水库存；v02 不把 `d lambda_m/dt` 转成额外气相水源项。
8. 增加膜水源项的平滑/限幅审计，但限幅必须有物理解释，不能只是数值截断。
9. 膜水动态状态必须反馈到膜电导率和欧姆损失。
10. v02 暂不新增液态水动态状态，只保留气相饱和裁剪和冷凝量审计。

## 拟新增参数

建议追加到 `StackParam` 末尾，避免破坏已有索引：

- `tau_memb_s`：膜水活度一阶动态时间常数，先作为敏感性扫描参数，不直接参与电压拟合。
- `k_mem_eff`：有效跨膜传输系数，初值建议 `0.1`，后续由无 EGR 与 CEGR 工况共同标定。
- `a_memb_min`：等效膜水活度下限，建议 `0`。
- `a_memb_max`：等效膜水活度上限，建议 `1`。
- `membrane_dynamic_mode`：膜水模式开关，`0` 为 v01 准静态，`1` 为 v02 零维动态。

### D. 参数扫描和步长敏感性

v02 不做 `tau_memb_s x k_mem_eff` 全组合大扫描，采用分层扫描：

1. 固定 `k_mem_eff = 0.1`，扫描 `tau_memb_s = 10 / 30 / 60 / 105 s`，29 个工况全跑。
2. 根据 `mMem_eff` 稳定性和量级，选择 1 到 2 个较好的 `tau_memb_s`。
3. 固定所选 `tau_memb_s`，扫描 `k_mem_eff = 1.0 / 0.5 / 0.2 / 0.1 / 0.05`，29 个工况全跑。
4. 不先改求解器和固定步长；只选择 3 个代表工况额外跑 `dt_s = 0.05 s`：
   - 低电流无 EGR
   - 高电流无 EGR
   - CEGR 高回流点
5. 若 `mMem_eff_last_range` 或 `lambda_m_last_range` 对步长高度敏感，再进入求解结构修正。

### E. 电压参数

v02 不同步调整电压拟合参数。

审计只输出 `E`、`etaAct`、`etaOhm`、`etaCon`、`V_cell`、`V_error` 和 `lambda_m -> etaOhm -> V_cell` 链路。等 `mMem/lambda_m/RH/冷凝` 稳定后，再进入独立电压标定阶段。

## 验收标准

1. 29 个工况能完成仿真，不出现负库存、NaN、Inf。
2. 相邻模块接口一致性继续全部通过。
3. 主要验收口径为末段动态指标，不以最后一个采样点单独判断稳定性。
4. 末段窗口建议采用最后 `20 s`，并输出：
   - `mMem_eff_last_mean`
   - `mMem_eff_last_range`
   - `mMem_eff_sign_changes_last`
   - `lambda_m_last_mean`
   - `lambda_m_last_range`
   - `a_memb_last_range`
   - `RH_ca_raw_last_max`
   - `condensed_ca_last_sum`
   - `|mMem_eff|/mH2O_prod` 末段均值和最大值
5. 末段 `mMem_eff` 的正负切换显著减少；若仍切换，末段范围应比 v01 明显下降，优先目标是降低到与产水量同量级。
6. 低电流工况 `|mMem_eff| / mH2O_prod` 不应再出现几十倍量级，除非有明确吸湿/脱湿瞬态解释。
7. 电压不作为第一验收目标，但 `lambda_m -> etaOhm -> V_cell` 链路必须可审计。
8. 无 EGR 与 CEGR 工况必须同时纳入审计，不能只让某一类工况稳定。

## Grillme 已收敛结论

1. 第 7 状态固定为 `a_memb`，`lambda_m` 是派生量。
2. `tau_memb_s` 先做 `10 / 30 / 60 / 105 s` 敏感性扫描，不直接参与电压拟合。
3. 引入 `k_mem_eff`，但它只约束跨膜水通量量级，不能用电压误差单独选择。
4. v02 不新增液态水动态状态；若 v02 后 RH、饱和裁剪或冷凝量仍跳变，再进入 v03 液态水库存。
5. 膜水动态用当前时步进入电堆核心时的气相状态推进；水蒸气饱和裁剪仍在 provisional 温度更新之后。
6. `a_memb` 是零维等效膜水活度滞后状态，不是严格膜水库存；v02 只输出等效膜水库存变化诊断。
7. 反扩散源项继续用 `lambda_ca - lambda_an`，半膜通量只做诊断。
8. 参与膜水计算的 `a_memb/a_ca/a_an` 限幅到 `[0,1]`，同时保留 raw 值做过饱和审计。
9. v02 以末段动态指标为主验收，最终单点值为辅。
10. v02 不先改求解器，只做代表工况步长敏感性。
11. v02 不同步调整电压拟合参数。
12. 参数扫描采用分层流程，不做 `tau_memb_s x k_mem_eff` 全组合大扫描。

## 当前结论

本次查阅 `02_PEMFC系统级建模与仿真` 后，v02 最合理路线是：

```text
v01 准静态膜水平均值
-> v02 零维膜水活度一阶动态 + 有效跨膜传输尺度审计
-> 若仍不稳定，再考虑液态水库存或更细的 CL/GDL 分层
```

不建议把 1D 穿膜模型直接塞入当前零维堆核心。
