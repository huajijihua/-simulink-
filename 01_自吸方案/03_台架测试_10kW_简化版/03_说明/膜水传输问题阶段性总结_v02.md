# 膜水传输问题阶段性总结 v02

本文档整理当前简化版 10 kW PEMFC/CEGR 台架模型中，围绕膜水传输问题已经完成的排查、修正、验证和阶段性判断。后续若继续做模型标定、液态水动态、局部传质损失或 CEGR 自增湿机理分析，应以本文件作为当前膜水模块的口径说明。

## 1. 问题背景

电堆核心模型按系统级零维模型组织，核心状态包括阴极氧气、阴极氮气、阴极水蒸气、阳极氢气、阳极水蒸气和电堆温度。质量守恒源项包括进出堆流量、电化学反应消耗/产水、跨膜水传输和水蒸气饱和后的冷凝裁剪。

第一轮修正 v01 已经把膜水反扩散从气相水蒸气浓度差改为膜水等效浓度差，并把电渗拖曳、反扩散、产水和饱和裁剪纳入详细审计。但 v01 详细审计显示：

- 全部 29 个工况在末段仍出现 `mMem` 正负切换。
- `mMem` 被符号/范围判据标记为不稳定的工况数为 29。
- 最大末段 `mMem` 范围约 `0.00191878 kg/s`。
- 低电流工况下，瞬时跨膜水通量量级可能明显大于产水量和入口水蒸气量，说明膜水源项过硬。

因此，v01 的问题不是简单接口错误，而是准静态膜水代数项与气相水库存、CEGR 回流延迟和显式时步推进耦合后形成的数值振荡。

## 2. 原因判断

对问题的判断分为三类：

1. 方程口径问题：准静态 `lambda_m = 0.5*(lambda_ca + lambda_an)` 每个计算时步直接跟随气相水活度变化，膜本体没有自身动态缓冲。
2. 参数尺度问题：全堆有效膜面积直接乘以瞬时浓度梯度后，`mMemRaw` 量级偏大，尤其在低电流下会放大符号反复。
3. 求解耦合问题：当前简化模型存在电堆出口、回流、混合入口之间的显式耦合，过强的膜水源项会在末段形成来回振荡。

因此本轮不优先改求解器，而是先修正膜水模块的动态口径和通量尺度，再用步长敏感性判断是否仍存在求解结构问题。

## 3. 文献和建模依据

本轮没有直接套用穿膜方向一维动态模型。原因是当前电堆模型是系统级零维模型，没有膜厚方向网格，也没有催化层/GDL/膜界面液水分布状态。直接搬入一维模型会引入与当前状态维度不匹配的内部自由度。

当前采用的依据如下：

- 电渗拖曳和反扩散仍采用 PEMFC 常见物理表达：电渗拖曳由 `n_d*i/F` 决定，反扩散由膜水浓度梯度和膜内水扩散系数决定。
- `lambda_eq(a)`、`n_d(lambda)` 和 `D_w(lambda,T)` 采用书籍和系统建模资料中的经验关系。
- 零维膜水动态参考系统级模型思路：膜本体水活度不是瞬时等于两侧气相平均值，而是按时间常数向阴阳极平均水活度滞后。

因此，当前方法属于可解释的系统级降阶模拟方法，不是纯数值拟合；但其中 `k_mem_eff` 是等效参数，不是可直接查表的材料常数。

## 4. v02 采用的膜水模型口径

电堆状态由 6 维扩展为 7 维：

```text
[mO2; mN2; mV_ca; mH2; mV_an; T; a_memb]
```

其中 `a_memb` 是零维等效膜水活度滞后状态，不是严格膜内水库存。膜体含水量由：

```text
lambda_m = lambda_eq(a_memb)
```

得到，并继续进入：

- 电渗拖曳系数 `n_d`
- 膜内水扩散系数 `D_w`
- 膜电导率和欧姆损失 `etaOhm`

膜水状态更新为：

```text
a_memb_target_raw = 0.5*(a_ca_raw + a_an_raw)
a_memb_target     = clamp(0.5*(a_ca + a_an), 0, 1)
a_memb_next       = clamp(a_memb + dt/tau_memb_s*(a_memb_target - a_memb), 0, 1)
lambda_m          = lambda_eq(a_memb_next)
```

参与膜水计算的 `a_ca/a_an/a_memb` 限幅到 `[0,1]`，同时保留 raw 水活度用于过饱和审计。

跨膜水通量先按原始通量链路计算：

```text
J_drag  = n_d * I / (F*A_cell)
J_diff  = D_w / delta_mem * (C_mem_ca - C_mem_an)
mMemRaw = (J_drag - J_diff) * A_cell * N_cell * M_H2O
mMem    = k_mem_eff * mMemRaw
```

符号约定保持不变：

```text
mMem > 0 表示水从阳极传到阴极
```

进入阴极和阳极气相水蒸气质量方程时，膜水项保持等量反号。随后仍执行气相水库存限幅和饱和蒸汽压冷凝裁剪，避免水蒸气质量变成负值或超过当前温度下的饱和上限。

## 5. 关键参数含义

当前默认参数为：

```text
tau_memb_s = 10
k_mem_eff  = 0.05
a_memb_min = 0
a_memb_max = 1
membrane_dynamic_mode = 1
```

`tau_memb_s` 表示等效膜水活度响应阴阳极气相水活度变化的时间常数。它不是电压拟合参数，主要用于描述膜水状态相对气相边界的动态滞后。

`k_mem_eff` 表示有效跨膜传输尺度系数。它不改变 `J_drag - J_diff` 的方向，只缩放进入气相质量方程的净跨膜水通量。它的物理解释是：当前零维模型没有显式描述催化层、GDL、界面吸脱附、局部液态水和有效传输面积变化，因此用一个小于等于 1 的等效系数约束总通量量级。

`k_mem_eff` 不能单独用电压误差选择，必须同时满足以下约束：

- `mMem` 末段不应持续正负跳变。
- `|mMem|/mH2O_prod` 不能在低电流下长期处于不合理的大倍数。
- 跨膜水通量方向应由阴阳极水活度和电流共同决定，而不是由数值限幅决定。
- 不能破坏阴极/阳极水源项等量反号的一致性。

## 6. 参数扫描结果

先固定 `k_mem_eff = 0.1`，扫描 `tau_memb_s`：

| tau_memb_s | 29 点末段 mMem 符号切换总数 | 最大 mMem 末段范围 kg/s | 平均 abs(mMem)/产水量 |
|---:|---:|---:|---:|
| 10 | 3196 | 4.3116e-4 | 0.98992 |
| 30 | 3194 | 4.3571e-4 | 1.5326 |
| 60 | 3400 | 4.8821e-4 | 2.4170 |
| 105 | 3402 | 5.0629e-4 | 2.4548 |

再固定 `tau_memb_s = 10`，扫描 `k_mem_eff`：

| k_mem_eff | 29 点末段 mMem 符号切换总数 | 最大 mMem 末段范围 kg/s | 平均 abs(mMem)/产水量 |
|---:|---:|---:|---:|
| 1.0 | 5800 | 2.4956e-3 | 3.3568 |
| 0.5 | 5800 | 2.4032e-3 | 3.3381 |
| 0.2 | 4200 | 8.5980e-4 | 2.9332 |
| 0.1 | 3196 | 4.3116e-4 | 0.98992 |
| 0.05 | 0 | 2.8366e-5 | 0.19507 |

扫描结论是：单纯增大 `tau_memb_s` 不能解决 `mMem` 振荡；真正敏感的是跨膜水通量进入气相库存的有效尺度。当前选择 `k_mem_eff = 0.05`，是因为它同时满足稳定性和量级约束。

## 7. 最终审计结果

最终 120 s 审计覆盖 13 个无 EGR 初始台架点和 16 个 CEGR0608 工况点，共 29 个工况。

基础审计结果：

- 成功工况：`29 / 29`
- 失败工况：`0`
- 默认 `tau_memb_s`：`10 s`
- 默认 `k_mem_eff`：`0.05`
- 电压 RMSE：`0.0112792 V`

详细审计结果：

- 接口一致性失败：`0 / 319`
- 末段 `mMem` 符号切换工况：`0 / 29`
- 最大 `mMem` 末段范围：`2.8366e-5 kg/s`
- 最大 `lambda_m` 末段范围：`0.707673`
- 最大 `a_memb` 末段范围：`0.0230094`
- `mMem` 时序行数：`29000`

步长敏感性检查选取低电流无 EGR、高电流无 EGR 和高回流 CEGR 三个代表工况。`dt = 0.1 s` 与 `dt = 0.05 s` 下均无末段 `mMem` 符号切换，`mMem` 末段范围变化很小。因此当前问题不再表现为明显的固定步长敏感问题。

## 8. 阶段性结论

当前膜水模块可以作为系统级零维模型中的可解释降阶方法使用。

它不是纯数值强行拟合，原因是：

- 跨膜水方向仍由 `J_drag - J_diff` 决定。
- 电渗拖曳、反扩散、膜水含量和膜电阻之间保留了物理链路。
- `mMem` 在阴极和阳极水质量方程中保持等量反号。
- `a_memb` 与 `lambda_m` 进入欧姆损失链路，而不是只作为稳定性变量。
- 参数选择基于通量量级、末段稳定性和步长敏感性，而不是直接用电压误差反推。

但它也不是严格的膜内水库存模型，原因是：

- `a_memb` 是等效活度滞后状态，不是由膜内水质量守恒严格积分得到。
- `k_mem_eff` 是有效传输尺度参数，代表当前零维模型省略的界面、层内和局部分布影响。
- 当前 v02 不包含液态水库存、局部淹没、局部干膜和厚度方向水分布。

因此推荐表述为：

```text
当前膜水传输是有物理约束的零维等效膜水动态模型。
它适合支撑系统级 CEGR 自增湿、入口湿度、平均膜水和欧姆损失分析；
不适合单独解释局部液水堵塞、局部干膜或膜厚方向水分布。
```

## 9. 后续使用建议

后续模型标定时，应把膜水模块与以下数据一起约束：

- 无 EGR 13 个台架点的入口压力、出口压力、流量、电流、电压。
- CEGR0608 工况点的回流比例、流量、入口/出口湿度压力、温度和电压。
- 阴极压差随电流密度增大的趋势。
- CEGR 下入口湿度、氧分压、膜水、欧姆损失和浓差损失之间的链路。

不建议用单一电压误差继续调 `k_mem_eff`。更合理的顺序是：

1. 先固定当前膜水稳定口径。
2. 用无 EGR 工况校准基础电堆压降、进出流、热平衡和电压模型。
3. 再用 CEGR 工况检查回流湿度、入口氧分压和膜水变化趋势。
4. 若 CEGR 电压下降仍无法解释，再引入液态水风险指标、局部传质损失或额外浓差损失项。
